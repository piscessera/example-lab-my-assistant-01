description: Start the pre-development phase by creating requirements, tasks, and plans.
---
1. Ask the user for the application name and high-level requirements if not provided.
2. Create the necessary directory structure: `[app-name]/docs/analysis`, `[app-name]/docs/tasks`, `[app-name]/docs/plans`.
3. Create `[app-name]/docs/analysis/requirements.md` detailing the user's needs.
4. Create `[app-name]/docs/tasks/main-tasks.md` with a checklist for Pre-Development, Development, and Post-Development.
5. Create `[app-name]/docs/plans/implementation-plan.md` outlining the technical approach and file structure.
6. Ask the user for confirmation to proceed to the Coding phase.
